**Pairing rule:** `input/photo1_1.jpg` is paired with `target/photo1_1.jpg` (identical filenames across the two folders).

The dataset was generated by a separate preprocessing script that splits composite reference photos into paired crops. Each crop is **344 x 384 pixels**, RGB, JPEG.

This format — **(input image → target image)** — is **different from DPR's native (image + SH lighting → relit image)** format. You must decide and clearly justify how to adapt DPR's training loop to work with my image-pair format. Valid strategies include (not exhaustive):

- Estimating SH coefficients on-the-fly from the target image using DPR's own lighting prediction head, then training the relighting network to map `(input, SH_target) → target`.
- Replacing the SH-conditioning branch entirely with a target-image-conditioned branch (more invasive but closer to pix2pix-style supervision).
- Using the target image purely as a pixel-space reconstruction loss while still feeding SH lighting derived from the target.

**Pick the approach that is most faithful to DPR's original architecture while minimizing code changes**, and explain your choice in the README with reasoning.
